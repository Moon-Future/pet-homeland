# 测试数据清理与存储路径

## 新上传路径

新上传文件按用户 openid 归档：

```text
users/{openid}/avatars/
users/{openid}/pet-spaces/pending/covers/
users/{openid}/pet-spaces/{petSpaceId}/covers/
users/{openid}/pet-spaces/{petSpaceId}/memories/
```

说明：

- 创建宠物时还没有 `petSpaceId`，封面会先进入 `pending/covers`。
- 编辑宠物时已有 `petSpaceId`，封面进入对应宠物目录。
- 回忆图片进入 `users/{openid}/pet-spaces/{petSpaceId}/memories/`。
- 旧文件不会迁移，清理仍以数据库中的 `fileId` 为准。

## 管理云函数

### `cleanupUserMedia`

按用户收集并清理云存储文件。仅管理员可调用。

预览：

```js
wx.cloud.callFunction({
  name: 'cleanupUserMedia',
  data: {
    targetOpenid: 'USER_OPENID',
    dryRun: true
  }
})
```

实际删除：

```js
wx.cloud.callFunction({
  name: 'cleanupUserMedia',
  data: {
    targetOpenid: 'USER_OPENID',
    dryRun: false
  }
})
```

默认会收集：

- 用户头像 `users.avatarFileId`
- 宠物头像/封面 `pet_spaces.avatarFileId` / `coverFileId`
- 回忆图片 `media.fileId`

删除云文件后，会把对应 `media.status` 标记为 `deleted`。

### `deleteTestUser`

测试期软删除某个用户及其业务数据。仅管理员可调用。

预览：

```js
wx.cloud.callFunction({
  name: 'deleteTestUser',
  data: {
    targetOpenid: 'USER_OPENID',
    dryRun: true
  }
})
```

实际删除：

```js
wx.cloud.callFunction({
  name: 'deleteTestUser',
  data: {
    targetOpenid: 'USER_OPENID',
    dryRun: false
  }
})
```

实际删除会：

- 调用 `cleanupUserMedia` 删除云文件。
- 把 `users.status` 标记为 `deleted`。
- 把 `pet_spaces.status` 标记为 `deleted`。
- 把 `memories.status` 标记为 `deleted`。
- 把 `media.status` 标记为 `deleted`。

注意：

- 不允许删除当前管理员账号。
- 数据库采用软删除，不物理移除文档。
- 生产环境使用前建议先 `dryRun: true` 检查数量和文件列表。
