# 小程序发布上架待办

## 发布定位

首版建议定位为“宠物生活记录工具 / 宠物纪念与相册工具”，弱化社区和广场属性。

当前功能包含公开小窝、星空广场、分享访问、用户上传文字和图片，属于 UGC 风险较高的形态。若使用个人主体，建议首版减少公开广场权重；若要长期运营广场，优先准备个体工商户或企业主体。

## 主体与类目

* 确认小程序主体类型：个人 / 个体工商户 / 企业。
* 确认服务类目与实际功能一致。
* 首版介绍避免使用“社交平台”“交易”“领养撮合”“宠物医疗”等容易触发更高资质要求的表达。
* 若保留星空广场，审核备注中说明它是宠物记录公开展示，不提供交易、领养、医疗建议。

## 备案与基础配置

* 完成微信小程序备案。

* 检查正式环境数据库权限，云数据库建议保持“客户端不可直接读写”，关键操作走云函数。

## 隐私合规

* 配置《用户隐私保护指引》。

* 检查所有涉及隐私接口或组件的调用是否与隐私指引声明一致。

## 管理员审核方案

首版不单独开发后端管理系统，管理员在小程序“我的”页面进入人工审核。

### 角色权限

* 用户表增加角色字段，默认普通用户：

```js
role: 'user'
```

* 管理员账号先在数据库中手动设置：

```js
role: 'admin'
```

* 前期先用 `role` 判断即可，后续审核能力变复杂时再扩展：

```js
permissions: {
  reviewContent: true,
  manageReports: true,
  hideContent: true
}
```

### 我的页面入口

* “我的”页面根据当前用户 `role === 'admin'` 显示“人工审核”入口。
* 普通用户不展示该入口。
* 入口页面建议：

```text
pages/admin-review/index
```

### 审核页面范围

人工审核页首版处理三类内容：

* 待审核公开小窝。
* 待审核公开回忆。
* 用户举报。

### 云函数权限校验

不能只靠前端隐藏入口，所有管理员云函数必须二次校验 `role === 'admin'`。

建议新增云函数：

```text
getAdminReviewItems
reviewContent
hidePublicContent
resolveReport
```

这些云函数都必须先根据 `openid` 查询 `users` 集合，确认当前用户是管理员后才能继续。

### 审核动作

* 通过：`reviewStatus = 'approved'`
* 拒绝：`reviewStatus = 'rejected'`
* 隐藏：`reviewStatus = 'hidden'`
* 处理举报：`status = 'resolved'`

星空广场只展示：

```js
visibility: 'discover'
reviewStatus: 'approved'
```

## 分享与访客

* 已实现分享入口参数：
  * `viewPetSpaceId`
  * `source=share`
* 提审前需要验证：
  * `private` 小窝不显示分享按钮。
  * `share` 小窝可通过分享链接访问，但不进入星空广场。

  * 分享访客只能查看和轻互动，不能编辑和记录日常。

## 功能边界

* 不提供宠物医疗诊断或治疗建议。
* 不提供宠物交易、领养撮合、寄养撮合。
* 不提供付费会员、虚拟商品、打赏等支付功能，除非后续补齐对应资质和支付审核。
* AI 分析若上线，定位为陪伴记录总结，不输出医疗、训练、疾病判断等高风险建议。

## 正式环境数据库权限

云开发数据库集合权限统一设置为“所有用户不可读写”，客户端不直接读写业务数据，全部走云函数。

| 集合 | 数据权限设置 | 说明 |
| --- | --- | --- |
| `users` | 所有用户不可读写 | 用户档案、openid、角色字段只允许云函数读写 |
| `pet_spaces` | 所有用户不可读写 | 小窝资料、可见性、审核状态由云函数校验后返回 |
| `memories` | 所有用户不可读写 | 回忆内容和媒体 fileID 由 `getMemories` 按权限过滤 |
| `media` | 所有用户不可读写 | 媒体元信息和审核状态由回忆/审核云函数维护 |
| `interactions` | 所有用户不可读写 | 互动计数和限频只能通过 `interactPetSpace` 修改 |
| `feedbacks` | 所有用户不可读写 | 意见反馈只允许 `submitFeedback` 写入 |
| `reports` | 所有用户不可读写 | 举报记录只允许 `submitReport` 写入、管理员云函数处理 |

## 首版建议

* 首版主打私密记录、时间轴、相册、分享可见。
* 星空广场可保留，但必须配套内容审核和举报治理。
* 若使用个人主体，建议先关闭或弱化广场入口，等主体和治理能力更完整后再放大公开广场。
