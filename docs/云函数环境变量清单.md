# 云函数环境变量清单

> 适用范围：图片上传重构后的七牛直传链路
> 日期：2026-06-08

---

## 一、变量说明

### `QINIU_ACCESS_KEY`

- 来源：七牛控制台
- 用途：签发七牛上传 token、调用七牛管理接口

### `QINIU_SECRET_KEY`

- 来源：七牛控制台
- 用途：与 `QINIU_ACCESS_KEY` 配套使用

### `UPLOAD_GRANT_SECRET`

- 来源：项目自行生成，不是七牛提供
- 用途：给 `sessionGrant` 和 `petUploadGrant` 做 HMAC 签名
- 要求：随机高强度字符串，建议至少 32 字节

生成示例：

```bash
openssl rand -hex 32
```

示例值：

```txt
9f4a7c1d2e3b8a6f0c11d22e33f44a55bb66cc77dd88ee99ff00112233445566
```

## 二、云函数与环境变量对照表

| 云函数 | 需要配置的环境变量 | 原因 |
|---|---|---|
| `login` | `QINIU_ACCESS_KEY`、`QINIU_SECRET_KEY`、`UPLOAD_GRANT_SECRET` | 返回 `sessionGrant`，且用户更换头像后需要删除旧头像 |
| `getQiniuUploadToken` | `QINIU_ACCESS_KEY`、`QINIU_SECRET_KEY`、`UPLOAD_GRANT_SECRET` | 验证 grant，并签发七牛上传 token |
| `cleanupRefs` | `QINIU_ACCESS_KEY`、`QINIU_SECRET_KEY`、`UPLOAD_GRANT_SECRET` | 验证 grant，并删除当前会话未提交的上传对象 |
| `reservePetSpaceId` | `UPLOAD_GRANT_SECRET` | 签发创建宠物页使用的 `petUploadGrant` |
| `createPetSpace` | `UPLOAD_GRANT_SECRET` | 验证创建宠物页的 `petUploadGrant` 与预分配小窝 ID |
| `getPetUploadGrant` | `UPLOAD_GRANT_SECRET` | 签发编辑宠物/写记忆页使用的 `petUploadGrant` |
| `updatePetSpace` | `QINIU_ACCESS_KEY`、`QINIU_SECRET_KEY` | 宠物头像/封面替换后删除旧文件 |
| `updateMemory` | `QINIU_ACCESS_KEY`、`QINIU_SECRET_KEY` | 记忆编辑或删除时删除旧图片 |
| `cleanupUserMedia` | `QINIU_ACCESS_KEY`、`QINIU_SECRET_KEY` | 后台清理用户七牛对象 |

## 三、推荐配置值模板

```txt
QINIU_ACCESS_KEY=你的七牛AccessKey
QINIU_SECRET_KEY=你的七牛SecretKey
UPLOAD_GRANT_SECRET=你自己生成的随机32字节以上字符串
```

## 四、为什么需要 `UPLOAD_GRANT_SECRET`

之前没有这个变量，是因为旧链路直接依赖：

1. 微信登录态
2. 服务端查库
3. 直接签七牛 token

现在新增 `UPLOAD_GRANT_SECRET`，是因为上传授权改成了两层短期票据：

1. `sessionGrant`
2. `petUploadGrant`

这两个 grant 都是项目自己的业务授权数据，不是微信或七牛发的，所以必须由项目自己的服务端密钥签名和验签。

它的主要作用：

1. 去掉上传热路径查库
2. 防止前端伪造 `petSpaceId`
3. 让一页内多次上传复用同一份短期授权

---

## 五、是否固定

`UPLOAD_GRANT_SECRET` 应该长期固定保存，不应每次部署都变。

如果修改它，会发生：

1. 旧的 `sessionGrant` 全部失效
2. 旧的 `petUploadGrant` 全部失效
3. 用户需要重新登录或重新进入页面，拿到新的 grant 才能继续上传

不会受影响的内容：

1. 已经上传成功并入库的图片
2. DB 中已经保存的 `Ref`

结论：

1. 平时不要随便改
2. 把它当成长期环境变量保存
3. 只有在主动做密钥轮换时才修改

---

## 六、部署注意事项

1. 同一套环境里的相关云函数，`UPLOAD_GRANT_SECRET` 必须完全一致。
2. 不要把 `UPLOAD_GRANT_SECRET` 复用成七牛的 `QINIU_SECRET_KEY`。
3. 如果切换了 `UPLOAD_GRANT_SECRET`，要预期短时间内上传页面会提示“登录态已失效”或“上传授权已失效”。
4. 新增云函数 `cleanupRefs`、`getPetUploadGrant` 部署时不要漏配环境变量。
