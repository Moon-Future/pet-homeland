# 宠物 NFC 数字身份系统设计

## 当前阶段

先实现数字身份底座，不依赖 NFC 实物。每只宠物在创建时获得：

- 唯一数字身份编号
- 不可猜测的身份入口 token
- 线上档案
- NFC 铭牌绑定状态预留字段

## 对外编号

格式：

```text
XC-YYYY-随机6位数字
```

示例：

```text
XC-2026-482913
```

规则：

- 创建宠物时由云函数自动生成。
- 用户不可修改。
- 全平台唯一。
- 删除或进入纪念状态后不回收。
- 不使用递增序号，避免早期号码攀比和平台规模暴露。

## 数据字段

`pet_spaces` 新增字段：

```js
{
  identityNo: 'XC-2026-482913',
  identityCode: '482913',
  identityYear: 2026,
  identityToken: '8xK29aQpL7mN...',
  identityStatus: 'active',
  identityCreatedAt: Date,
  nfc: {
    status: 'unbound',
    tagId: '',
    boundAt: null
  }
}
```

字段说明：

- `identityNo`：用户可见的永久编号。
- `identityCode`：编号中的随机 6 位数字。
- `identityYear`：编号年份。
- `identityToken`：二维码/NFC 真实入口使用，不在普通公开档案中展示。
- `identityStatus`：身份状态，当前默认为 `active`。
- `nfc.status`：`unbound` / `bound` / `lost` / `replaced`。

## 入口

当前小程序入口页：

```text
/pages/identity/index?code=XC-2026-482913
/pages/identity/index?token=xxxxxxxxxxxxxxxxxx
```

后续 NFC 实体铭牌建议写入 token 入口，而不是直接写入手机号、宠物资料或数据库 ID。

## 云函数

- `createPetSpace`：创建宠物时生成 `identityNo` 和 `identityToken`。
- `getMyPetSpaces`：为旧宠物档案懒生成缺失的身份字段。
- `getPetSpaceDetail`：返回身份编号和 NFC 绑定状态。
- `resolvePetIdentity`：通过 `identityNo` 或 `identityToken` 解析数字身份入口。

## 页面展示

- 宠物主页展示“数字身份编号 / 档案类型 / 身份状态 / NFC 状态”。
- 宠物列表展示身份编号。
- 数字身份入口页展示基础档案，并可进入宠物档案页。
