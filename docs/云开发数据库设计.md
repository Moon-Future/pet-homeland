# 星宠乡云开发数据库设计

## 集合总览

MVP 先使用云开发数据库、云存储和云函数跑通真实闭环。客户端只负责采集资料和上传文件，身份确认、用户档案创建与更新通过 `login` 云函数完成。

## `users`

用户微信登录后的主档案。`openid` 是业务身份主键，建议为 `openid` 建唯一索引。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `_id` | string | 云数据库自动生成 |
| `openid` | string | 微信用户 openid |
| `unionid` | string | 存在开放平台绑定时写入 |
| `appid` | string | 当前小程序 appid |
| `nickname` | string | 用户昵称，来自昵称输入或微信资料授权 |
| `avatarUrl` | string | 当前展示头像，可为云存储 fileID |
| `avatarFileId` | string | 上传到云存储后的头像 fileID |
| `vip` | boolean | 会员标记，默认 `false` |
| `stats.petCount` | number | 用户宠物主页数量 |
| `stats.memoryCount` | number | 回忆数量 |
| `stats.mediaCount` | number | 媒体数量 |
| `stats.shareCount` | number | 分享次数 |
| `status` | string | `active` / `disabled` / `deleted` |
| `lastLoginAt` | Date | 最近登录时间 |
| `createdAt` | Date | 创建时间 |
| `updatedAt` | Date | 更新时间 |

## `pet_spaces`

宠物小窝或纪念馆。用户可以创建多条。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `_id` | string | 云数据库自动生成 |
| `ownerOpenid` | string | 创建者 openid |
| `petName` | string | 宠物昵称 |
| `petType` | string | `cat` / `dog` / `other` |
| `breed` | string | 品种 |
| `gender` | string | `male` / `female` / `unknown` |
| `lifeStatus` | string | `with_me` / `in_stars` |
| `birthDate` | string | 出生日期，格式 `YYYY-MM-DD`，可为空；`birthDate` 和 `arrivalDate` 至少填一个 |
| `arrivalDate` | string | 来到身边的日期，格式 `YYYY-MM-DD`，可为空；`birthDate` 和 `arrivalDate` 至少填一个 |
| `deathDate` | string | 离去日期，格式 `YYYY-MM-DD`，`in_stars` 状态必填 |
| `avatarUrl` | string | 宠物头像展示地址，可为云存储 fileID |
| `avatarFileId` | string | 宠物头像云存储 fileID |
| `coverUrl` | string | 封面图展示地址，可为云存储 fileID |
| `coverFileId` | string | 封面图云存储 fileID |
| `theme` | string | `cloud` / `rainbow` / `starry` / `sakura` |
| `story` | string | 一句话介绍或宠物故事 |
| `visibility` | string | `private` / `link` / `public` |
| `stats.missCount` | number | 想念次数 |
| `stats.memoryCount` | number | 回忆数量 |
| `stats.starCount` | number | 星光数量 |
| `stats.mediaCount` | number | 媒体数量，创建时上传封面则为 `1` |
| `stats.flowerCount` | number | 送花数量 |
| `status` | string | `active` / `archived` / `deleted` |
| `createdAt` | Date | 创建时间 |
| `updatedAt` | Date | 更新时间 |

## `memories`

宠物时间轴图文回忆。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `_id` | string | 云数据库自动生成 |
| `petSpaceId` | string | 关联宠物主页 |
| `ownerOpenid` | string | 创建者 openid |
| `title` | string | 回忆标题 |
| `content` | string | 正文 |
| `memoryDate` | string | 发生日期 |
| `type` | string | `birth` / `home` / `birthday` / `travel` / `daily` / `farewell` / `custom` |
| `mediaFileIds` | string[] | 图片或视频 fileID |
| `sortOrder` | number | 排序权重 |
| `status` | string | `active` / `hidden` / `deleted` |
| `createdAt` | Date | 创建时间 |
| `updatedAt` | Date | 更新时间 |

## `media`

云存储文件元信息，方便后续做相册、审核和资源清理。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `_id` | string | 云数据库自动生成 |
| `petSpaceId` | string | 关联宠物主页，可为空 |
| `ownerOpenid` | string | 上传者 openid |
| `fileId` | string | 云存储 fileID |
| `type` | string | `image` / `video` |
| `category` | string | `avatar` / `cover` / `album` / `memory` |
| `size` | number | 文件大小 |
| `width` | number | 图片宽度 |
| `height` | number | 图片高度 |
| `status` | string | `pending_review` / `active` / `blocked` / `deleted` |
| `createdAt` | Date | 创建时间 |

## `interactions`

纪念馆互动计数，按用户、日期、类型限频。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `_id` | string | 云数据库自动生成 |
| `petSpaceId` | string | 关联宠物主页 |
| `openid` | string | 互动用户 openid |
| `type` | string | `miss` / `flower` / `star` |
| `dateKey` | string | 日期键，例如 `2026-06-03` |
| `count` | number | 当日次数 |
| `createdAt` | Date | 创建时间 |
| `updatedAt` | Date | 更新时间 |

## 权限建议

云数据库权限建议设置为“所有用户不可读写”，登录、创建、编辑、删除等关键写入都走云函数。后续需要公开访问的纪念馆详情由 `getMemorialDetail` 云函数按 `visibility` 判断后返回。

云存储建议按目录区分：

| 目录 | 用途 |
| --- | --- |
| `users/avatars/` | 用户头像 |
| `pet-spaces/covers/` | 宠物主页封面 |
| `pet-spaces/albums/` | 相册图片 |
| `pet-spaces/memories/` | 时间轴媒体 |
