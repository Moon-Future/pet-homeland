# 星宠乡云开发数据库设计

## 集合总览

MVP 先使用云开发数据库、云存储和云函数跑通真实闭环。客户端只负责采集资料和上传文件，身份确认、用户档案创建与更新通过 `login` 云函数完成。

## `users`

用户微信登录后的主档案。`openid` 是业务身份主键，建议为 `openid` 建唯一索引。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `_id` | string | 云数据库自动生成 |
| `openid` | string | 微信用户 openid |
| `unionid` | string | 存在开放平台绑定时写入 |
| `appid` | string | 当前小程序 appid |
| `nickname` | string | 用户昵称，来自昵称输入或微信资料授权 |
| `avatarUrl` | string | 当前展示头像，可为云存储 fileID |
| `avatarFileId` | string | 上传到云存储后的头像 fileID |
| `vip` | boolean | 会员标记，默认 `false` |
| `role` | string | `user` / `admin`，默认 `user`；管理员账号可在数据库中手动改为 `admin` |
| `permissions` | object | 预留细粒度权限，首版可为空对象 |
| `stats.petCount` | number | 用户宠物主页数量 |
| `stats.memoryCount` | number | 回忆数量 |
| `stats.mediaCount` | number | 媒体数量 |
| `stats.shareCount` | number | 分享次数 |
| `status` | string | `active` / `disabled` / `deleted` |
| `lastLoginAt` | Date | 最近登录时间 |
| `createdAt` | Date | 创建时间 |
| `updatedAt` | Date | 更新时间 |

## `interactions`

小窝互动计数，按用户、日期、类型限频。主人每种互动每日最多 10 次，间隔 10 分钟；他人每种互动每日最多 1 次。小窝页按钮展示当前用户今日次数，累计次数展示在统计区。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `_id` | string | 云数据库自动生成 |
| `petSpaceId` | string | 关联宠物主页 |
| `openid` | string | 互动用户 openid |
| `type` | string | 陪伴中：`cuddle` / `feed` / `checkin`；已去星星：`miss` / `flower` / `star` |
| `dateKey` | string | 日期键，例如 `2026-06-03` |
| `count` | number | 当日次数 |
| `isOwner` | boolean | 是否主人本人互动 |
| `lastInteractedAt` | Date | 最近一次互动时间 |
| `createdAt` | Date | 创建时间 |
| `updatedAt` | Date | 更新时间 |

## 权限建议

云数据库权限建议全部收紧为“所有用户不可读写”。小程序客户端不直接读写数据库，登录、创建、编辑、删除、公开展示、举报和管理员审核全部走云函数。云函数在服务端拥有读写权限，并按 `openid`、小窝归属、`visibility`、`reviewStatus` 和管理员 `role` 做二次校验。

在云开发控制台的数据库“数据权限”页，以下集合建议逐个设置为：

| 集合 | 控制台数据权限选项 | 原因 |
| --- | --- | --- |
| `users` | 所有用户不可读写 | 包含 openid、昵称、头像、角色权限和统计数据，只能由 `login`、管理员校验等云函数读写 |
| `pet_spaces` | 所有用户不可读写 | 包含宠物资料、公开状态和审核字段；公开读取必须走 `getPetSpaceDetail` / `getDiscoverPetSpaces` 按可见性与审核状态过滤 |
| `memories` | 所有用户不可读写 | 包含用户文字和媒体 fileID；读取必须走 `getMemories`，非主人只能看已通过审核内容 |
| `media` | 所有用户不可读写 | 保存云存储文件元信息和审核状态，写入与状态更新由回忆/审核云函数处理 |
| `interactions` | 所有用户不可读写 | 互动限频和计数必须走 `interactPetSpace`，避免客户端篡改次数 |
| `feedbacks` | 所有用户不可读写 | 意见反馈包含用户填写内容，只允许 `submitFeedback` 写入，后续由管理员或云端查看 |
| `reports` | 所有用户不可读写 | 举报记录包含举报人和处理结果，只允许 `submitReport` 写入、管理员云函数读取和处理 |

若控制台暂时只看到“仅创建者可读写”等选项，不要为这些业务集合选择开放读写；首版统一选择“所有用户不可读写”。截图中的 `feedbacks`、`interactions`、`media`、`memories`、`pet_spaces`、`users` 都应设置为该选项；新增 `reports` 后也同样设置。

## 内容安全和人工审核

写入链路先走自动内容安全，再决定是否进入人工审核：

| 场景 | 自动审核 | 人工审核 |
| --- | --- | --- |
| 创建/编辑 `private` 小窝 | 文本 + 图片 | 不进入，`reviewStatus = 'not_required'` |
| 创建/编辑 `share` 小窝 | 文本 + 图片 | 不进入，`reviewStatus = 'not_required'` |
| 创建/编辑 `discover` 小窝 | 文本 + 图片 | 进入 `pending_review` |
| `private` / `share` 小窝新增或编辑回忆 | 文本 + 图片 | 不进入，`reviewStatus = 'not_required'` |
| `discover` 小窝新增或编辑回忆 | 文本 + 图片 | 进入 `pending_review` |
| 意见反馈 | 文本 | 不进入 |
| 举报 | 文本 | 进入管理员举报处理列表 |

当小窝从 `private` / `share` 切换为 `discover` 时，系统会把该小窝下尚未人工审核通过的回忆批量改为 `pending_review`；已人工通过且保留 `reviewedAt` 的回忆不重复送审，`hidden` 内容也不会被批量恢复。小窝从 `discover` 改回非公开时，尚未完成公开审核的回忆回到 `not_required`，不再出现在管理员待审列表。

`checkContentSecurity` 云函数需要部署，并在云函数配置中声明 OpenAPI 权限：

```json
{
  "permissions": {
    "openapi": [
      "security.msgSecCheck",
      "security.imgSecCheck"
    ]
  }
}
```

人工审核与治理建议区分：

* `not_required`：当前不需要人工公开审核，通常用于未公开小窝和未公开回忆。
* `approved`：内容通过审核，可公开展示。
* `rejected`：待审内容未通过，用户修改后可重新提交。
* `hidden`：管理员治理下架，用于已经公开或被举报的异常内容；隐藏后访客不可见，管理员可恢复。

云存储建议按目录区分：

存储额度策略：首版限制每个账号最多上传 `30` 张回忆图片，头像与小窝封面不计入；后续可在 `getMediaQuota` / `addMemory` / `updateMemory` 中按会员等级扩展额度。

| 目录 | 用途 |
| --- | --- |
| `users/avatars/` | 用户头像 |
| `pet-spaces/covers/` | 宠物主页封面 |
| `pet-spaces/albums/` | 相册图片 |
| `pet-spaces/memories/` | 时间轴媒体 |
