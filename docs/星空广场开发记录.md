# 星空广场开发记录

## 页面定位

星空页从原来的 mock 星空改为“宠物广场”：

* 随机遇见不超过 20 只公开小窝宠物。
* 支持刷新重新获取一片星空。
* 点击宠物头像先选中并在底部展示资料，再次点击头像或底部资料卡进入对应宠物小窝。
* 进入他人小窝后只能查看和轻互动，不能编辑或记录日常。

## 已完成

### 星空广场

* `miniprogram/pages/star-space/` 改为真实广场页面。
* 支持筛选：
  * 全部星星
  * 最近更新
  * 正在星宠乡
  * 住在星星上
  * 猫咪
  * 狗狗
  * 其他
* 支持“换一片星空”刷新。
* 星空中最多展示 20 只宠物。
* 陪伴中和已离去用不同状态文案、颜色和星光样式区分。
* 底部只展示当前选中的宠物资料卡，避免 20 条数据形成过长列表。
* 加入加载星点骨架屏。
* 空状态引导用户去设置自己的小窝公开到星空广场。
* 星点使用预置分散布局，降低头像重叠概率。

### 隐私与可见性

* 创建/编辑小窝时支持设置可见性：
  * 仅自己可见
  * 通过分享可见
  * 出现在星空广场
* 新建小窝默认仍为“仅自己可见”。
* `getDiscoverPetSpaces` 只返回明确设置为“出现在星空广场”的小窝。
* `getPetSpaceDetail` 按可见性校验访问权限。
* `getMemories` 按小窝可见性校验时间轴、相册、回忆详情访问权限。
* `interactPetSpace` 和 `getPetInteractionSummary` 对私密小窝访客访问做拦截。

### 云函数

* 新增 `cloudfunctions/getDiscoverPetSpaces`。
  * 返回随机宠物小窝列表。
  * 支持 `all` / `recent` / `with_me` / `in_stars` / `cat` / `dog` / `other` 筛选。
  * 仅从 `visibility=discover` 的 active 小窝中抽取。
* 新增 `cloudfunctions/getPetSpaceDetail`。
  * 支持按 `petSpaceId` 读取单个小窝。
  * 返回当前访问者是否为主人。
  * 私密小窝仅主人可访问，分享/广场小窝可被访客查看。
* 调整 `cloudfunctions/getMemories`。
  * 支持访客查看有权限的小窝时间轴、相册和回忆详情。
* 调整 `cloudfunctions/interactPetSpace`。
  * 增加“留爪印”互动类型。
  * 记录互动来源：星空广场、分享访问、个人访问、主人详情。
* 调整 `cloudfunctions/getPetInteractionSummary`。
  * 返回今日互动计数。
  * 主人可看到今日访客数和访客轻互动次数。

### 访客模式

* 小窝详情页支持 `viewPetSpaceId` 访客入口。
* 星空广场点击宠物时写入 `viewPetSpaceId`，再切换到小窝 tab。
* 访客模式下：
  * 隐藏编辑按钮。
  * 陪伴中小窝展示贴贴、喂食、留爪印。
  * 已去星星小窝展示想你了、送花、点亮星光。
  * 可查看最近回忆、时间轴和相册。
  * 不可记录今天、编辑资料或编辑日常。
  * 互动后展示更具体的反馈文案。
* 从首页或宠物列表进入自己的小窝时会清理访客缓存。

## 待做

### 内容安全

* 公开广场正式上线前需要接入头像、昵称、故事、回忆内容审核。
* 意见反馈和公开小窝需要后台处理入口。
* 可考虑增加公开小窝举报入口、隐藏入口和运营审核状态。

### 后续体验优化

* 分享链接进入小窝时把互动来源标记为 `share`。
* 广场可继续增加“今天有人更新”“新朋友”等运营筛选。
* 主人侧可增加最近访客列表，但需要先确认隐私边界和展示粒度。

## 注意事项

当前星空广场已经从 MVP 的 active 小窝随机抽取，升级为只展示用户明确公开到广场的小窝。正式发布前仍必须补齐内容审核与举报/后台处理链路，避免公开内容缺少治理入口。
