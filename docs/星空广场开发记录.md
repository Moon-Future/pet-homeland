# 星空广场开发记录

## 页面定位

星空页从原来的 mock 星空改为“宠物广场”：

* 随机遇见不超过 20 只宠物。
* 支持刷新重新获取一片星空。
* 点击宠物头像先选中并在底部展示资料，再次点击头像或底部资料卡进入对应宠物小窝。
* 进入他人小窝后只能查看和轻互动，不能编辑或记录。

## 已完成

### 星空广场

* `miniprogram/pages/star-space/` 改为真实广场页面。
* 支持筛选：
  * 全部星星
  * 正在星宠乡
  * 住在星星上
* 支持“换一片星空”刷新。
* 星空中最多展示 20 只宠物。
* 陪伴中和已去星星用不同状态文案、颜色和星光样式区分。
* 底部只展示当前选中的宠物资料卡，避免 20 条数据形成过长列表。

### 云函数

* 新增 `cloudfunctions/getDiscoverPetSpaces`。
  * 返回随机宠物小窝列表。
  * 支持 `all` / `with_me` / `in_stars` 筛选。
  * 当前 MVP 从 active 小窝中抽取。
* 新增 `cloudfunctions/getPetSpaceDetail`。
  * 支持按 `petSpaceId` 读取单个小窝。
  * 返回当前访问者是否为主人。
* 调整 `cloudfunctions/getMemories`。
  * 支持访客查看未删除小窝的时间轴、相册和回忆详情。

### 访客模式

* 小窝详情页支持 `viewPetSpaceId` 访客入口。
* 星空广场点击宠物时写入 `viewPetSpaceId`，再切换到小窝 tab。
* 访客模式下：
  * 隐藏编辑按钮。
  * 陪伴中小窝只展示贴贴、喂食。
  * 已去星星小窝展示想你了、送花、点亮星光。
  * 可查看最近回忆、时间轴和相册。
  * 不可记录今天、编辑资料或编辑日常。
* 从首页或宠物列表进入自己的小窝时会清理访客缓存。

## 待做

### 隐私与可见性

* 增加小窝可见性设置：
  * 仅自己可见
  * 通过分享可见
  * 出现在星空广场
* `getDiscoverPetSpaces` 后续应只返回明确开启“出现在星空广场”的小窝。
* `getPetSpaceDetail` 和 `getMemories` 后续应根据可见性校验访问权限。

### 广场体验

* 增加“最近更新”筛选。
* 增加星空加载骨架屏。
* 支持按宠物类型筛选：猫、狗、其他。
* 增加空状态引导用户公开自己的小窝。
* 星点位置可以根据宠物数量动态避让，减少重叠。

### 轻互动增强

* 访客互动后展示更有仪式感的反馈文案。
* 小窝主人可看到“今天有几位朋友来过”。
* 增加“留个爪印”互动类型。
* 增加互动来源：星空广场、分享访问、个人访问。

### 内容安全

* 公开广场上线前需要接入头像、昵称、故事、回忆内容审核。
* 意见反馈和公开小窝需要后台处理入口。

## 注意事项

当前 MVP 为了快速验证广场玩法，`getDiscoverPetSpaces` 从 active 小窝中随机抽取。正式发布前必须补齐隐私开关和权限校验，避免用户未授权的小窝出现在公开广场。
