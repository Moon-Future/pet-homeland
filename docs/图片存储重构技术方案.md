# 图片存储重构技术方案

> 状态：待评审 → 待实施
> 日期：2026-06-07
> 背景：项目处于开发阶段，无线上数据。将图片存储从微信云开发存储迁移到七牛云，并预留后期切换到腾讯云 COS 的扩展点。

---

## 一、目标

1. 把"存储 provider"做成可替换层，今天接七牛、后期能切腾讯云 COS。
2. **首期只实现七牛 provider**，不保留 wx-cloud 兼容代码、不保留旧 `cloud://` 字段。
3. 因为无线上数据，所有冗余 / 兼容字段直接删除，schema 一步到位。
4. 路径不暴露 `openid`，改用项目自生成的 `uid`。

---

## 二、存储对象引用结构（Ref）

所有图片在 DB / 接口中统一表达为 `Ref` 对象。**DB 只存以下三个字段，URL 不入库**：

```js
{
  storage: 'qiniu',          // provider 标识，后期可扩 'tencent-cos'
  bucket:  'cl8023',         // 七牛 bucket 名
  key:     'project/star-pet-village/users/{uid}/avatars/1733xxxxxx-abc.jpg'
}
```

**字段说明**

| 字段 | 含义 | 备注 |
|---|---|---|
| `storage` | provider 类型 | 当前固定 `'qiniu'` |
| `bucket` | 存储空间名 | 当前固定 `'cl8023'`；不同 provider 语义略不同（COS 是 bucket-appid） |
| `key` | 对象路径（不带域名） | 包含 prefix `project/star-pet-village/` |

**`url` 为什么不入库**：

- `url = cdnHost + '/' + key` 完全由 `key` 推导，是冗余字段。
- CDN 域名一旦变更（换域名 / 加二级域名 / 临时降级），只需改一行配置；存了 url 就要批量改 DB。
- 接口下行时由服务端拼好 url 挂到响应里（具体见下文 §六）。

## 四、用户 ID（uid）

### 4.1 为什么不用 openid

- openid 是微信生态内的用户唯一标识，理论上应作为后端内部数据使用，**不应出现在对象存储路径里**（路径会被 CDN log / 截图等场景泄露）。
- 后期可能接入苹果登录、手机号登录、扫码登录等，openid 不是统一身份标识。
- uid 是项目自有 ID，与登录方式解耦。

### 4.2 uid 规则

- **格式**：12 位 base62（字符集 `0-9 A-Z a-z`），如 `Kx3pQ9aB2mNs`。
- **生成时机**：用户首次登录时（`login` 云函数中创建 users 文档时），同步生成并写入。
- **唯一性**：12 位 base62 ≈ 3.2 × 10²¹ 种组合，碰撞概率可忽略；生成后查重一次（防御性）。
- **不可猜测**：使用 `crypto.randomBytes` 而非 `Math.random`。
- **索引**：users 集合的 `uid` 字段建唯一索引（云开发控制台 → 数据库 → 索引管理手动建）。

### 4.3 users 集合 schema 调整

```diff
{
  openid: String,
+ uid:    String,   // 12 位 base62，创建时生成，唯一
  ...
}
```

### 4.4 后端流程

`getQiniuUploadToken` 内部：

```
openid (来自 cloud.getWXContext)
  → 查 users 表得到 uid
  → 拼 key: project/star-pet-village/users/{uid}/...
  → 签 token
```

> 多一次 DB 查询，可接受。后期可加内存缓存（先不做）。

## 五、上传路径规范

key 由服务端 `getQiniuUploadToken` 生成，规则：

```
{keyPrefix}/users/{uid}/{subPath}/{timestamp}-{random}.jpg
```

其中 `subPath` 按图片类型决定：

| imageType | subPath 规则 |
|---|---|
| `avatar` | `avatars` |
| `petCover` | `pet-spaces/{petSpaceId}/covers`；无 petSpaceId 时 `pet-spaces/pending/covers` |
| `petAlbum` | `pet-spaces/{petSpaceId}/albums`；无 petSpaceId 时 `pet-spaces/pending/albums` |
| `memory` | `pet-spaces/{petSpaceId}/memories`；无 petSpaceId 时 `pet-spaces/pending/memories` |

`petSpaceId` 在拼接前需 sanitize：仅保留 `[A-Za-z0-9_-]`，截断 80 字符。

### 5.1 关于"一万用户就有一万个目录"

七牛 / COS / S3 等对象存储**没有"目录"概念**，所有 key 都是平铺存的，斜杠 `/` 只是 key 字符串里的普通字符。控制台看到的"文件夹"只是按 `/` 做的视觉分组。

- 一万 / 一百万 / 一亿个 key 性能完全相同（O(1) 哈希）。
- 计费按存储量 + 流量 + 请求数，与 key 数量无关。
- 按用户分前缀的**真正好处在运维侧**：删用户 / 排查 / 审计 / 后期拆库都能按 prefix 批量操作，对存储引擎本身零开销。

## 六、上传 & 读取流程

### 6.1 上传

```
小程序                       云函数(getQiniuUploadToken)              七牛
  │                                  │                                  │
  │ ① uploadImage({type,             │                                  │
  │   petSpaceId, filePath})         │                                  │
  │ ───────────────────────────────► │                                  │
  │                                  │ ② 校验 openid                    │
  │                                  │   openid → uid                   │
  │                                  │   生成 key                        │
  │                                  │   qiniu SDK 签 putPolicy          │
  │                                  │   (scope=bucket:key,             │
  │                                  │    mimeLimit=image/*,            │
  │                                  │    fsizeLimit=10MB,              │
  │                                  │    expires=3600)                 │
  │ ③ { uploadToken, key }           │                                  │
  │ ◄─────────────────────────────── │                                  │
  │                                                                     │
  │ ④ wx.uploadFile                                                     │
  │   url:      https://upload-z2.qiniup.com                            │
  │   filePath: <crop 结果>                                              │
  │   name:     'file'                                                  │
  │   formData: { token, key }                                          │
  │ ──────────────────────────────────────────────────────────────────► │
  │                                                                     │
  │ ⑤ { hash, key }                                                     │
  │ ◄────────────────────────────────────────────────────────────────── │
  │                                                                     │
  │ ⑥ 客户端组装 ref = { storage, bucket, key }（不含 url）              │
  │   传给业务云函数（createPetSpace / addMemory / login）入库            │
```

### 6.2 读取（接口下行）

业务读取云函数（`getMyPetSpaces` / `getMemories` 等）从 DB 拿到 ref 后，在响应里**额外**挂上 url 字段（不入库）：

```js
// DB 里存的
{ avatarRef: { storage, bucket, key } }

// 接口下行响应
{
  avatarRef: { storage, bucket, key },
  avatarUrl: 'https://qiniu.cdn.cl8023.com/' + ref.key   // 服务端拼接
}
```

前端 wxml 仍绑定 `{{item.avatarUrl}}`，无需感知 ref 结构。

async function upload({ type, petSpaceId, filePath }) {
  const { result } = await wx.cloud.callFunction({

  }

### 9.4 `cloudfunctions/getQiniuUploadToken/index.js`（伪代码）

```js
const cloud = require('wx-server-sdk')
const qiniu = require('qiniu')
const crypto = require('crypto')

cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

const BUCKET     = 'cl8023'
const KEY_PREFIX = 'project/star-pet-village'

exports.main = async (event = {}) => {
  const { OPENID: openid } = cloud.getWXContext()
  if (!openid) return { ok: false, message: '无法获取微信登录态' }

  const type = sanitizeType(event.type)
  if (!type) return { ok: false, message: '不支持的图片类型' }

  // openid → uid（用户首次登录时已生成）
  const userResult = await db.collection('users').where({ openid }).limit(1).get()
  const uid = userResult.data && userResult.data[0] && userResult.data[0].uid
  if (!uid) return { ok: false, message: '用户不存在或缺少 uid' }

  const subPath = buildSubPath(type, event.petSpaceId)
  const random  = crypto.randomBytes(6).toString('hex')
  const key     = `${KEY_PREFIX}/users/${uid}/${subPath}/${Date.now()}-${random}.jpg`

  const mac = new qiniu.auth.digest.Mac(
    process.env.QINIU_ACCESS_KEY,
    process.env.QINIU_SECRET_KEY,
  )
  const putPolicy = new qiniu.rs.PutPolicy({
    scope:      `${BUCKET}:${key}`,
    expires:    3600,
    fsizeLimit: 10 * 1024 * 1024,
    mimeLimit:  'image/*',
  })
  const uploadToken = putPolicy.uploadToken(mac)

  return { ok: true, uploadToken, key }
}
```

### 9.5 `login` 云函数中生成 uid（伪代码）

```js
const crypto = require('crypto')

function generateUid() {
  const alphabet = '0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz'
  const bytes = crypto.randomBytes(12)
  let uid = ''
  for (let i = 0; i < 12; i += 1) {
    uid += alphabet[bytes[i] % 62]
  }
  return uid
}

async function generateUniqueUid() {
  for (let attempt = 0; attempt < 5; attempt += 1) {
    const uid = generateUid()
    const existing = await db.collection('users').where({ uid }).limit(1).get()
    if (!existing.data || !existing.data.length) return uid
  }
  throw new Error('uid 分配失败')
}

// 新用户创建分支中：
const uid = await generateUniqueUid()
const user = {
  openid,
  uid,
  // ... 其他字段
}
```

### 9.6 `cloudfunctions/_shared/storage.js`（每个云函数复制一份）

```js
const qiniu = require('qiniu')

const BUCKET   = 'cl8023'
const CDN_HOST = 'https://qiniu.cdn.cl8023.com'
const ZONE     = qiniu.zone.Zone_z2   // 华南-广东

function buildUrl(refOrKey) {
  if (!refOrKey) return ''
  const key = typeof refOrKey === 'string' ? refOrKey : refOrKey.key
  return key ? `${CDN_HOST}/${key}` : ''
}

function buildBucketManager() {
  const mac = new qiniu.auth.digest.Mac(
    process.env.QINIU_ACCESS_KEY,
    process.env.QINIU_SECRET_KEY,
  )
  const cfg = new qiniu.conf.Config({ zone: ZONE })
  return new qiniu.rs.BucketManager(mac, cfg)
}

// refs: Ref[]
async function deleteObjects(refs) {
  const valid = (refs || []).filter((r) => r && r.storage === 'qiniu' && r.key)
  if (!valid.length) return

  const bm  = buildBucketManager()
  const ops = valid.map((r) => qiniu.rs.deleteOp(r.bucket || BUCKET, r.key))

  // 分批（七牛单次 batch 上限 1000）
  for (let i = 0; i < ops.length; i += 500) {
    await new Promise((resolve) => {
      bm.batch(ops.slice(i, i + 500), () => resolve())
    })
  }
}

module.exports = { buildUrl, deleteObjects }
```

## 十、安全要点

1. **SecretKey 仅放云函数环境变量**，绝不下发到小程序。
2. `getQiniuUploadToken` 强制校验 `openid` 存在，key 路径强制带 `users/{uid}/`，并通过 putPolicy 的 `scope = bucket:key` 限定**用户只能上传到指定 key**。
3. `fsizeLimit` 10MB、`mimeLimit` `image/*`、`expires` 3600s。
4. 删除接口（`cleanupUserMedia`）维持仅管理员可调用。
5. uid 使用 `crypto.randomBytes`，不可预测；不直接暴露 openid。
